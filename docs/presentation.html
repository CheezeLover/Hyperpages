How it works

From question to insight in seconds

Hyperset connects your data, your LLM, and your team through a secure, self-hosted pipeline.

1

User authenticates via Caddy

Your team logs in at auth.your-domain.com. Caddy issues a JWT and injects identity headers (X-Webauth-*) into every request. No passwords are ever sent to Superset directly.

2

Portal loads with Superset embedded

The Next.js portal renders the split-pane UI. Superset opens in an iframe using header-based SSO. The service column gives access to chat, custom pages, and settings.

3

User asks a question in natural language

The chat API sends the message to your configured LLM (OpenAI-compatible). The system prompt includes Superset context, knowledge base chunks, and tool definitions.

4

AI calls tools via MCP

The LLM invokes tools over the Model Context Protocol: execute SQL, list charts, inspect datasets, create visualizations. HMAC-signed tokens secure portal-to-MCP communication.

5

Results stream back to the user

The AI response streams in real-time. Tool results appear as rich reference cards. New charts open in Superset automatically. Follow-up suggestions help guide the next question.

Security

Designed for sensitive data environments

Every layer is built with a zero-trust mindset. Your data never leaves your infrastructure.

🔐

Header-based SSO

Superset never sees user passwords. Caddy injects identity headers after JWT verification. All inter-service traffic stays on the internal hyperset-net network.

🔑

HMAC-signed Tokens

Portal-to-MCP communication is secured with HMAC signatures. Per-session tokens expire and cannot be replayed, preventing unauthorized tool invocations.

🛡️

Role-based Access Control

Groups and roles flow from your auth provider through JWT claims. Custom pages, admin settings, and knowledge base management all enforce role checks independently.

⏱️

Rate Limiting

Each session is limited to 20 chat messages per 60 seconds. Agentic tool-call loops are bounded by a configurable max-turns parameter to prevent runaway API usage.

🔒

Encrypted Sessions

User sessions use iron-session with encrypted, HMAC-verified cookies. No sensitive data is stored client-side in plaintext.

📦

Zero External Dependencies

No telemetry, no analytics beacons, no external API calls are made by the platform itself. The only outbound traffic is to your own configured LLM endpoint.

Your data stays yours
